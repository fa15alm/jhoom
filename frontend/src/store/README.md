# Store

This folder is reserved for frontend state that needs to be shared across screens.

Good first stores/contexts to add when connecting the backend:

- `auth` - token, current user, login/logout.
- `profile` - profile details and settings.
- `logs` - daily health logs and calendar summaries.
- `healthPlan` - onboarding answers and generated plan.
- `social` - friends, posts, comments, and pending requests.
